Laborator 5

Exerciții de antrenament

Se consideră un tip de date pentru expresiile aritmetice:

Inductive AExp :=
| number : nat -> AExp
| plus : AExp -> AExp -> AExp
| div : AExp -> AExp -> AExp.

Fie funcția de evaluare a unei expresii eval, care returnează 0 în cazul în care expresia de evaluat este div a b (a / b), iar b = 0: 

Fixpoint eval (e : AExp) : nat :=
  match e with
  | number n => n
  | plus e1 e2 => (eval e1) + (eval e2)
  | div e1 e2 => if Nat.eqb (eval e2) 0
                   then 0
                   else Nat.div (eval e1) (eval e2)
  end.
  1. (Din laboratorul precedent) Demonstrați prin inducție că orice expresie e, eval e >= 0.

Vom demonstra mai întâi următoarea proprietate (conform cerinței, ea este prin inducție):

Lemma nats_are_positive : forall n, n >= 0.
Proof.
  induction n; unfold ge in *.
  - SearchPattern (_ <= _).
    apply Nat.le_refl. (* reflexivity for <= *)
  - SearchPattern (_ <= _).
    Check Nat.le_le_succ_r.
    apply Nat.le_le_succ_r. (* prepare for IH application*)
    apply IHn. (* use the inductive hyp *)
Qed.

Această proprietate este de fapt deja demonstrată și în Coq:

Check Nat.le_0_l.
Nat.le_0_l
     : forall n : nat, 0 <= n

Acum, demonstrația cerută de exercițiu este foarte ușoară:

Lemma eval_greater_than_0 :
  forall e, eval e >= 0.
Proof.
  intros.
  SearchPattern (_ <= _).
  apply nats_are_positive. (* sau Nat.le_0_l *)
Qed.
  1. Definiția lui eval nu este tocmai corectă: matematic, 3 / 0 nu este egal cu 0. În astfel de situații este bine să returnăm un rezultat cât mai precis. Deoarece pentru numere naturale rezultatul evaluării 3 / 0 nu produce un nat, vom crea un nou tip denumit ErrorNat care să includă o valoare de eroare pentru aceste situații:
Inductive ErrorNat :=
| err : ErrorNat
| num : nat -> ErrorNat.

Acum putem putem defini adunarea peste ErrorNat:

Fixpoint plus_ErrorNat (n1 n2 : ErrorNat) : ErrorNat :=
  match n1, n2 with
  | err, _ => err
  | _, err => err
  | num v1, num v2 => num (v1 + v2)
  end.

Exerciții propuse:

  1. (0.5 pct.) Scrieti o funcție care implementează împărțirea peste ErrorNat:
Fixpoint div_ErrorNat (n1 n2 : ErrorNat) : ErrorNat :=
  match n1, n2 with
    ...
  end.

Apoi, utilizând plus_ErrorNat și div_ErrorNat completați funcția eval de mai jos:

Fixpoint eval (e : AExp) : ErrorNat :=
  match e with
  | number n => ...
  | plus e1 e2 => ...
  | div e1 e2 => ...
  end.

  1. (0.5 pct.) Adăugați la sintaxa limbajului IMP discutat la curs (vezi fișierul: imp.v) instrucținea if. Scrieți măcar trei programe în limbajul IMP care să utilizeze această instrucțiune. Spre exemplu, puteți scrie programe care calculează maximul/minimul a trei numere, sau programe care compară două numere și returnează valori cu o anumită semnificație (e.g., 0 - dacă numerele sunt egale, 2 - dacă primul număr este mai mic decât al doilea, etc.).

  2. (1 pct.) Modificați funcția eval din definiția limbajului IMP discutat la curs (vezi fișierul: imp.v) astfel încât să puteți rula programele ce conțin instrucțiunea if scrise la exercițiul 2. Testați programele scrise.

Bonus: 4. (1 pct.) Formulați și verificați proprietăți ale programelor utilizând mecanismele de demonstrare puse la dispoziție de Coq. Pentru detalii, cereți ajutorul profesorului de laborator.
